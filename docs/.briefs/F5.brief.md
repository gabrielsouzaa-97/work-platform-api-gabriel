## O que está BOM (reconhecimento explícito — exigência do método auditoria-senior)

Sprint F5 demonstra ótimas práticas a serem preservadas:

- **Defensive design — translator-first**: `$upstreamVerbTokens = $this->translator->cmdToCliArgv($cmd);` é a primeira instrução de `LifecycleAsyncAction::execute()`. Verbs bloqueados disparam exception ANTES de consultar cluster, criar idempotency key, persistir job ou audit log. Test prova invariante via 3 contadores `->toBe(0)`.
- **Negative invariants nos testes**: `!in_array('users:create')`, `!in_array('--async')`, `!in_array('--json')`, `!argsContainConsecutive(['user', 'delete'])`, `!argsContainConsecutive(['group', 'delete'])`. Estes guards são o oposto exato do bug-simétrico que escondeu ISSUE-006.
- **Helper `argsContainConsecutive`**: abstração elegante — desacopla a assertion da posição absoluta (slug precede verbo, flags seguem) sem perder rigor.
- **`BlockedOnUpstreamException` design**: separação semântica clara de `UnknownVerbException`. Uma vira HTTP 501 (intencional/feature pendente), outra é bug do caller (HTTP 500). Carrega `cmd` readonly para diagnóstico estruturado.
- **Decision #ARCH-4 thoroughness**: alternativa B vs A pesada honestamente; rationale "coesão por origem > coesão por destino" defendável; consequences nomeia a porta de saída quando upstream D3/D4 entregar membership (`remover de BLOCKED_ON_UPSTREAM` — controllers/Livewire não mudam).
- **Skill `vocabulary-translator` rewrite**: lista os 3 vocabulários (não 2), referencia Decision #ARCH-4, codifica regra "NUNCA injetar `cmd_canonical` diretamente no argv". Protege contra repetição do bug.
- **Idempotency hash adjustment** acompanha mudança de contrato: novo shape `[$username]` (sem email) gera hash diferente, mantém 24h dedup window saudável.
- **CSV consolidation**: `apps:enable/disable` agora dispara 1 SSH round-trip + 1 row em `jobs`/`idempotency_keys`, recuperando atomicidade (ISSUE-006 §DP2). Tests cobrem single-app (`'contacts'`) e multi-app (`'calendar,contacts,mail'`).
- **stdin payload schema** (`users:create`): `{password, email?, groups?}` — email/groups deixam de ser positional silent no-op upstream.
- **Branch isolation**: feature branch `uds/fix/lifecycle-async-cmd-argv` criado ANTES do primeiro Write — respeita branch protection (lição registrada em AGENT-NOTES.md em 2026-05-20).
- **Opt-in contract test em diretório separado**: `tests/Contract/` com Pest config próprio. Acertou o que o `Feature/`+`RefreshDatabase` quebrava.
