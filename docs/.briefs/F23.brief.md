## Causa raiz (CQ-N46-001 — HIGH)

`LifecycleAsyncAction` persiste `job_type` via `JobTypeTranslator::cmdToJobType()`:

| Comando OCC (cmd) | Persistido no Job | Projector N46 aceita? |
|---|---|---|
| `groups:create` | `group_create` | **Não** — só `groups:create` |
| `groups:delete` | `group_delete` | **Não** — só `groups:delete` |

`TenantGroupProjector` compara `job_type` com listas que usam o vocabulário **cmd** (`groups:*`), não o vocabulário **persistido** (`group_*`). Resultado: webhook terminal de sucesso **nunca** chama upsert/delete em `tenant_groups` até sync manual.

O padrão correto já existe em `TenantUserProjector`:

```php
USER_CREATE_TYPES = ['user_create', 'users:create'];
USER_DELETE_TYPES = ['user_delete', 'users:delete'];
```

## Estratégia de correção

### Núcleo (F23.1) — espelhar TenantUserProjector

1. Em `TenantGroupProjector`, alinhar constantes:
   - `GROUP_CREATE_TYPES = ['group_create', 'groups:create']`
   - `GROUP_DELETE_TYPES = ['group_delete', 'groups:delete']`
2. Corrigir `WebhookTenantGroupProjectionTest` para fixtures com `group_create`/`group_delete` (caminho real).
3. Manter alias `groups:*` para compatibilidade / regressão.
4. Ideal: teste integrado Lifecycle → webhook → assert row (fecha o falso verde CQ-N46-002).

### Validação UI/API (F23.2–F23.3)

- Membership no OccPanel: lookup case-insensitive idêntico a `TenantGroupMembership` (API).
- `createGroup`/`deleteGroup`: aplicar `regex:/^[a-zA-Z0-9._\- ]+$/` já presente em `CreateGroupRequest`.
- Bloquear nome reservado `admin` (case-insensitive) em `CreateGroupRequest` **e** OccPanel (membership já bloqueia atribuir `admin`, mas create não).

### UX pós-fix do projector (F23.4)

Após F23.1, espelhar o fluxo de users: poll do job terminal + `loadGroups()` no success (create e delete). Sem F23.1, poll não atualiza o read model.

### Sync operacional (F23.5)

- Incrementar `TenantGroupSyncReport.updated` quando row existente for atualizada (ou remover campo se métrica for abandonada — preferir incrementar).
- `tenant-groups:sync`: exit `FAILURE` se qualquer customer falhar (cron alerta).
