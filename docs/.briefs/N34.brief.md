## Objetivo

Implementar saga de onboarding composta exposta em `/api/v1/onboarding`: uma chamada inicia provision → readiness → admin → apps → branding (opcional); polling via `GET /v1/onboarding/{id}`; idempotência 24h; falha parcial com retry seguro e runbook (compensação automática **fora** de escopo — ADR R-5).

## Gate de sucesso (ADR Fase 4)

1. Signup completo por **1 POST** + polling GET — resposta inicial **202** em <2s
2. Replay idempotente **não duplica** tenant (mesmo `onboarding_id`)
3. Etapa falha expõe estado `partial`/`failed` com retry seguro por step
4. Branding bloqueado por D-02 → step `skipped` + `capability_not_available` honesto (sem vocabulário NC)
5. `correlation_id` propagado (`onboarding_id` → `job_id`) — infra N32.6
6. Runbook `docs/runbooks/onboarding-saga.md` com game-day de saga parcial
7. Spec externo + CONTRACTS-V1 alinhados; CI redocly + testes verdes

## Saga steps (sequência)

```
provision_tenant → wait_readiness → create_admin → enable_apps → set_branding (opcional)
```

- Steps assíncronos retornam `job_id`; saga avança após webhook/polling terminal
- `set_branding`: se upstream bloqueia (D-02) → step `skipped`, saga pode completar em `completed` com warning no step
