## Quality Constraints

1. **FQDN server-side obrigatório (N39.1)** — Normalização `trim → strtolower → rtrim('/')` em `prepareForValidation()` do FormRequest E no Livewire `updatedDomain()` — proibido confiar só no preview client. Trailing slash **strip** (não 422). Regex `/^[a-z0-9-]+(\.[a-z0-9-]+)+$/` após normalize. Rejeitar 422 apenas protocolo (`://`) e formato inválido. `ProvisionTenantRequest` herda via `ProvisionCustomerRequest`.

2. **OccPassthroughService como único hop SSH na UI (N39.2)** — Listar users via `exec($customer, 'user:list', ['--json'])` — não duplicar argv `occ-exec`. Parse JSON defensivo. Timeout wall-clock **30s** em `loadUsers()`.

3. **Poll com throttle e timeout (N39.3, N39.4)** — `wire:poll` condicional; `JobLogFetcher` max 1 call/15s por job_id (cache); poll user-create timeout wall-clock ~120s. Padrão existente: `jobs/show.blade.php:3-5`.

4. **Readiness sem alterar promote/timeout (N39.5)** — **Decisão: AuditLog** `action=customer_readiness_probe` com `{attempt, error, probe}` a cada falha (sem migration em `customers`). Não alterar transição `provisioning_finishing → active/failed` de `ProbeCustomerReadinessJob`.

5. **Retrofit visual sem mudança funcional (N39.6 stretch)** — Substituir `<style>` hex em `customers/*` por tokens M3 (`bg-surface-container`, `text-on-surface`) — referência `cluster-servers/index.blade.php`.

6. **Cluster remove com guarda (N39.7 stretch)** — Soft-delete só se zero customers ativos no `cluster_server_id`; confirmação por nome; gate `manage-cluster-servers`.

## Technical Advisory

1. **Incidente trailing slash** — FQDN com `/` final quebra Traefik `Host()` e readiness probe (404). Prioridade máxima N39.1.

2. **Senha NC33** — Painel valida `>= 8`; NC rejeita com `>= 10` + dicionário. Alinhar hint sem reimplementar política upstream.

3. **SSH load durante poll** — N39.4 tail log pode amplificar chamadas SSH; throttle 15s é obrigatório, não opcional.
