## Fase 1 — Deploy

### 1.1 Registrar SHA de partida

No clone local:

```bash
git fetch origin main
git rev-parse origin/main
```

Anotar o SHA (ex.: `267b6ea…`) — será o alvo do deploy.

### 1.2 SSH e atualizar código

```bash
ssh operador@api.lab.mework360.com.br
cd /opt/mework360-deployer

git fetch origin
git checkout main
git pull origin main
git rev-parse HEAD    # deve coincidir com SHA anotado
```

### 1.3 Build e subir containers

```bash
docker compose build --target production
docker compose up -d
docker compose ps     # todos healthy
```

### 1.4 Migrations e cache

```bash
docker compose exec app php artisan migrate --force
docker compose exec app php artisan migrate:status   # nenhuma Pending

# Somente se APP_KEY já está no .env do LAB:
docker compose exec app php artisan config:cache
docker compose exec app php artisan route:cache
```

### 1.5 Health check pós-deploy

```bash
curl -sf https://api.lab.mework360.com.br/up
docker compose logs --tail=50 app
```

| Check | Esperado |
|-------|----------|
| `GET /up` | HTTP 200 |
| Containers | `app`, `nginx`, `database`, `redis` healthy |
| Logs app | Sem stack trace / fatal error recente |

**Critério de saída Fase 1:** SHA deployado registrado + `/up` 200.

## Fase 2 — Smoke job async

### 2.1 Disparar job de teste

Tenant canário: **`qa-platform-lab`** (`qa-platform-lab.lab.mework360.com.br`).

Escolher **uma** opção (preferir A):

**A — Painel (recomendado):**

1. Login em `https://api.lab.mework360.com.br` → Customers → tenant **`qa-platform-lab`**
2. OCC panel → `users:create` (email/senha de teste)

curl -sS -X POST "https://api.lab.mework360.com.br/api/customers/qa-platform-lab/users" \

## Fase 3 — Verificar `jobs.summary` (MariaDB)

```bash
docker compose exec app php artisan tinker --execute="
  \$j = App\Models\Job::where('job_id', '<JOB_ID>')->first();
  echo 'status=' . \$j?->status . PHP_EOL;
  echo 'exit_code=' . var_export(\$j?->exit_code, true) . PHP_EOL;
  echo 'summary=' . (\$j?->summary ? 'POPULATED' : 'NULL') . PHP_EOL;
"
```

Ou SQL direto no MariaDB:

```sql
SELECT job_id, status, exit_code,
       CASE WHEN summary IS NULL THEN 'NULL' ELSE 'POPULATED' END AS summary_state,
       LENGTH(summary) AS summary_bytes
FROM jobs
WHERE job_id = '<JOB_ID>';
```

| Campo | Esperado (F10.3 OK) |
|-------|---------------------|
| `status` | `finished` ou `success` |
| `summary` | **POPULATED** — JSON com linhas de log |
| `exit_code` | `0` (ou preenchido via `JobLogFetcher` se webhook veio null — ISSUE-013) |

**Se `summary` NULL com job SUCCESS:** falha F10.3 — verificar logs `storage/logs/laravel.log` por `JobLogFetcher` / exit 101.
